Lightweight Website Initiative

Count every bit or every bit counts

What is a lightweight website?

A lightweight website strives to be efficient and slim. It uses the minimal necessary html, css, js, in order to render and deliver content.

Why does it matter?

Every bit has multiple costs. There is the initial cost of the server to process the request and generate the response. The response is then transmitted through the internet to the client’s browser. And finally the browser must process and render the response.

Transmission takes time, which is especially important for more limited baandwidths. Either a speed limit, for a slower connection speed, or a capacity in the case of a transfer limit like 5GB per month. There is also an electricity cost with transmitting the bit.

Server processing to generate the response takes time and electricty. The more bits required for the response the longer for the server to build it and start sending it.

The client browser receiving the response also must process and render it. Every bit in the response gets processed and impacts the display in some way.

All these costs add up. They may individually be infintessmally small, needing many thousands and hundreds of thousands of bits to add up to a single cent. This adds up for every request from every user throughout the day. Overtime it is a measurable amount.

Even if the total remains small it is still a savings which can help reduce global energy usage.

Email

The same applies to email. More and more email is becoming an html document in and of itself. There are far fewer emails than http requests but the same conditions apply. Every email must be downloaded, processed, and rendered. I’ve received 100KB emails which had fewer than a single line of informative text in them. All those bits could have been avoided.

Internet Citizenship

We are all citizens of the internet. As such we have a responsibility for stewardship of both what we publish and what we consume.

Alternatives

There are alternatives, like the gopher protocol. It is not as widespread and I do not think switching completely to any of the alternatives is viable. Consider supporting multiple while keeping an http presense as well.

Am I a curmudgeon?

Probably. Many people have “unlimited” data and “high-speed” internet connections. Some people pay for every bit they download. And everyone pays for processing and rendering the content, through the electricity required.
Your webpage is costing someone else money. I think we should all be considerate of those consuming what we publish.

Instead of piling more into a page, and adding embedded videos, javascript effects and interactions, aall requiring additional libraries which must be processed as well, save some bits. Every bit saved adds up and if every page and every email saved even just one bit that would equate to measurable savings across the vast amount of requests generated every minute.

Answers

There is no one right answer. This concept and this page has faults. If we could all just try alittle it would still make a difference.
